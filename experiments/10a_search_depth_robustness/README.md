# Experiment 10a README

## Experiment id

`10a_search_depth_robustness`

## Research question

Does deeper symbolic search reveal a new interpretable residual structure, or does it produce syntactically complex formulas that approximate the same smooth/polynomial residual curve more verbosely?

## Current role

This experiment feeds the search-depth robustness discussion. It is a limited-scope diagnostic on two exemplar high-c corpora, Shakespeare and War and Peace. It supports the claim that depth-10 search produces syntactic complexity without a new simple mechanistic expression, but it does not establish the polynomial decomposition across all 25 English corpora.

## Outputs produced

### `outputs/step10_ablation_per_corpus.csv`

Two rows, one for Shakespeare and one for War and Peace. Records the step-2 expression, saved depth-10 expression, grid diagnostics for `(step10 - step2)`, polynomial R² values for degrees 3-5, and degree-5 coefficients.

### `outputs/polynomial_decomposition_per_corpus.csv`

Two rows. Records ZM, step-2, step-10, and direct `ZM + step-2 + polynomial` RMSE values for degrees 3-5.

### `outputs/poly_transfer.csv`

Two rows. Records the saved zero-shot degree-5 polynomial transfers: Shakespeare -> War and Peace and War and Peace -> Shakespeare.

### `outputs/aggregate_statistics.csv`

Claim-facing aggregate rows for the degree-5 R² values, direct polynomial RMSE comparisons, saved transfer RMSE values, and missing-transfer claim count.

### `outputs/manifest.json`

Machine-readable inventory of source bundles, output schemas, satisfied claim rows, and blocked claim rows.

### `BLOCKED.md`

Documents transfer claims present in manuscript prose but absent from the saved historical bundle.

## Methods

No depth-10 search or polynomial fitting is rerun. This migration consolidates saved historical outputs.

`zipf_step10_ablation` is canonical for depth-10 expression trees and polynomial approximability of `(step10 - step2)` on the fixed grid. `zipf_head_poly_decomposition` is canonical for direct residual-decomposition RMSE comparisons. `zipf_head_poly_transfer` is canonical for the two saved transfer rows.
